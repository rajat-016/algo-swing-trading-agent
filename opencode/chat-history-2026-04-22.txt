## Issue 1: Trading Mode Fixed to .env Priority

**Problem:** User set TRADING_MODE=live but server still ran in paper mode

**Root Cause:**
- run.ps1 line 78 sets `$env:TRADING_MODE = $Mode` (default "paper") before starting Python
- Pydantic loads OS environment variables FIRST, then .env file values

**Fix (config.py):**
- Added `_read_env_file_value()` function to read .env directly at module import time
- At module load, read .env file value and force into `os.environ["TRADING_MODE"]`
- This ensures .env file ALWAYS takes priority over run.ps1 env var

## Issue 4: Historical Data Returns 0 Candles

**Problem:** Historical data returned 0 candles in live mode

**Root Causes:**
1. Symbol format mismatch - ChartInk uses full names, Zerodha uses short symbols
2. Wrong API endpoint format
3. BSE vs NSE conflict

**Fix (zerodha.py + chartink.py):**

1. **ChartInk update**: Read Symbol column directly

2. **Symbol mapping**: Added proper caches:
   - `_name_to_symbol_cache`: ChartInk name → Zerodha symbol
   - `_symbol_to_token_cache`: Zerodha symbol → instrument token (NSE only)

3. **API endpoint**: Fixed to correct path format:
   - Before: `/instruments/historical?instrument_token=X&interval=Y`
   - After: `/instruments/historical/{token}/{interval}`

4. **NSE priority**: Filter to only use NSE EQ instruments

## Issue 5: ML Model Integration

**Problem:** Need ML-based momentum ranking instead of pure technical

**Solution:**
- Created `train_model.py` to train ML model
- Based on reference project from algo-trading-platform
- Uses GradientBoostingClassifier
- Trained model: model.joblib (93.47% accuracy)

**Hybrid Approach:**
- If ML BUY probability > 0.1 → use ML score
- Otherwise → fall back to technical score
- This handles imbalanced training data (mostly HOLD)
